1. Auto-Initialize (Simplest Method)

This is the recommended approach for most pages:

<script type="module">
  import { autoInit } from '/js/navigation/index.js';
  autoInit();
</script>

2. Manual Initialize with Callbacks

<script type="module">
  import { initNavigation } from '/js/navigation/index.js';

  const nav = await initNavigation({
    componentPath: '/components/navigation.html',
    mountSelector: 'body',
    onLoad: (navInstance) => {
      console.log('Navigation loaded!', navInstance);
    },
    onError: (error) => {
      console.error('Navigation error:', error);
    }
  });
</script>

3. Using the Navigation Class

<script type="module">
  import { Navigation } from '/js/navigation/index.js';

  const nav = new Navigation();
  await nav.init();

  // Access sub-modules
  const mobileMenu = nav.getMobileMenu();
  const keyboardNav = nav.getKeyboardNav();
</script>

Keyboard Navigation Testing

Test the following keyboard shortcuts:

  • Arrow Left/Right - Navigate between top-level menu items
  • Arrow Down - Open dropdown or navigate to next dropdown item
  • Arrow Up - Close dropdown or navigate to previous dropdown item
  • Enter/Space - Activate current item
  • Escape - Close all dropdowns or mobile menu
  • Home - Jump to first menu item
  • End - Jump to last menu item
  • Tab - Natural tab behavior (closes dropdowns)

Accessibility Features

  • WCAG 2.1 AA compliant keyboard navigation
  • Proper ARIA attributes (aria-expanded, aria-current, aria-label)
  • Focus trap in mobile menu and dropdowns
  • Screen reader announcements via live regions
  • Skip to main content link
  • Clear focus indicators

Mobile Menu Features

  • Body scroll lock when menu is open
  • Focus trap keeps focus within menu
  • Closes on Escape key
  • Closes on click outside
  • Smooth animations with CSS classes
  • Auto-closes when resizing to desktop size
